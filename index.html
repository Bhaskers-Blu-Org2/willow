Please report any bugs you find using this Connect template. In particular, if you receive an unexpected error during installation, we want to hear about it as soon as possible! Please attach the generated log file, if one exists, to the bug so we can further diagnose.

Other known issues and limitations are listed below:

General

  • Many common project types, including ASP.NET Core and Universal Windows Platform are not supported in the first preview release; these will be added after //build.
  • Install sizes reported are close estimates, but may be slightly inaccurate.
  • The new setup engine does not yet register file associations for project and solution types that it handles, so opening from File Explorer will not instantiate Visual Studio “15” Preview at this time.
  • The Markdown TextMate bundle does not correctly colorize headings.
  • Source file colorization may not work correctly when debugging code loaded from a source control server.
  • When you install just the Visual Studio core editor component, Team Explorer commands that require Office integration (send mail and open in Excel) will be inoperable. These features work if you have any other component installed.
  • In the current build of the installer, the Modify command operates by uninstalling and reinstalling, rather than directly modifying the installation. This is a temporary limitation of the setup engine and will be fixed in a later build.
  • Uninstall fully removes Visual Studio, but some external dependencies (e.g. Windows SDK, .NET Framework SDK) may still be present on your machine. You can manually remove these from Add/Remove Programs if they are not being used by other Visual Studio releases you may have on your machine.

.NET Desktop Development

  • No other known issues with this component.

C++

  • The lightweight installer does not currently support ATL or MFC.
  • The C++ component will not work correctly in a side-by-side installation with an Visual Studio “15” Preview install from the classic installer. This is a known issue and will be fixed in the next release.

Python

  • PyLint is not currently supported.

Unity

  • To use the Unity tools in Visual Studio, you will first need to install the latest Unity beta, which can be found at this location: http://unity3d.com/unity/beta.
